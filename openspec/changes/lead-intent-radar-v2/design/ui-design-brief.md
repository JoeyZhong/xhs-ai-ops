## 2. 五处增量（逐条给「改什么 / 在哪 / 需要你定的设计点」）

### Delta 1 · 队列卡加「信源标识」（必做）
- **现状**：队列卡 `.stat-tag` 行固定写「小红书 · @作者 · 检测延迟」。
- **V2**：信源有三种——小红书 / 知乎 / 猪八戒。每张卡要能一眼看出来自哪个池子。
- **你定**：
  - 信源用 **线性图标** 还是 **文字标签**？放在 `row1`（与触发场景 badge 同行）还是仍在 `.stat-tag`？
  - 是否加一排 **信源筛选**（全部/小红书/知乎/猪八戒）。注意顶栏右侧已有「待处理/已触达/跳过」三 Tab，信源筛选要与它**视觉分层不打架**（建议独立的小分段控件或下拉，别和生命周期 Tab 混成一排）。

### Delta 2 · 猪八戒结构化 meta（必做，但露出程度你定）
- **背景**：猪八戒「需求单」自带结构化字段——**预算 / 交付周期 / 是否已接单**（存在 `lead.meta`，仅猪八戒有；小红书/知乎为空）。
- **你定**：
  - 卡片上要不要露（比如一颗「预算 ¥X」小徽章）？还是只在详情区？
  - 详情区给猪八戒线索加一块 meta（预算/周期/接单状态）小卡，还是塞进「原帖」卡内？
  - 字段缺失（小红书/知乎线索没有 meta）时这块**不出现**，不要留空槽。

### Delta 3 · 详情区主按钮「按信源切换」（必做，本单最关键）
详情区操作按钮组（基线 `.actions`）按 `lead.source` 呈现两种形态：

| 信源 | 主按钮 | 次按钮 | 「打开原帖」链接文案 |
|---|---|---|---|
| **小红书** | **一键发送**（primary，brand 实底）+ 状态见 Delta 4 | 改 / 跳过 / 复制草稿+打开原帖（兜底降级） | 打开原帖 |
| **知乎** | **复制草稿 + 打开问题**（沿用 V1 primary） | 改 / 跳过 | 打开问题 |
| **猪八戒** | **复制草稿 + 打开需求单**（沿用 V1 primary） | 改 / 跳过 | 打开需求单 |
- 「原帖」卡的来源行（基线写死「小红书 · X 分钟前发布」）与卡头链接文案，也要随源变（小红书帖/知乎问题/猪八戒需求单）。
- 首触草稿卡头（基线「首触草稿 · 审计报告快出（人设口吻）」）可标注**文体**：小红书短回复 / 知乎专业回答 / 猪八戒接单报价（人设同一个，文体不同）。
- **你定**：xhs 详情里「一键发送」与「复制+打开原帖（兜底）」如何并存——主次？还是引擎未就绪时才显示兜底？

### Delta 4 · 小红书「一键发送」状态反馈（必做，仅 xhs）
一键发送是 V2 写端，逐条人工点击触发。给这一个按钮设计出**全部状态**的视觉：

| 状态 | 触发条件 | 视觉表达（你定具体样式） |
|---|---|---|
| 可发送 | 校验通过 + 引擎就绪 + 未超限 | brand 实底主按钮「一键发送」 |
| 演练（dryrun，**默认态**） | 引擎未切到真发 | 主按钮可点但明示「当前为演练 · 不会真实发出」（如按钮副标/旁注 chip）。这是**当前默认**，要让用户清楚没真发。 |
| 发送中 | 点击后 | loading 态、禁重复点 |
| 已发 | 成功 | 成功态 + 可显示平台返回（如评论 id / 时间）；该条转「已触达」 |
| 校验未过禁发 | 引流词/雷同未过 | 按钮 disabled + 原因（沿用 V1 引流词/雷同度两枚校验 chip 的红/绿） |
| 引擎未就绪 | 配 reajason 但凭证无效 | 禁发 + 「引擎未就绪」提示，回退到「复制+打开原帖」 |
| 速率超限 | 当日已发 ≥5 | 禁发 + 「今日 5/5 已达上限 · 距下次可发 X 分钟」（速率锁定 ≤5 条/天、间隔 5–15min 随机，见 proposal §8-B） |
- **你定**：状态是放按钮内（变文案/变色/带 spinner），还是按钮旁一行状态条 + 计数（今日 N/5）。

### Delta 5 · 顶部 stats 是否按源拆分（可选）
- 现状 4 格：今日合格线索 / （V1 mockup 里是平均检测延迟）/ 本周沟通机会·北极星 / 本周成交。
- **你定**：是否在某格加「按源分布」（如今日合格 7：红书4·知乎2·八戒1 的小三段）。能不动就不动，北极星保持单列高亮。

## 3. 硬约束（必须遵守）

- **无 emoji**：全部用线性 SVG / lucide 风格图标（基线 HTML 的 `svg{stroke:currentColor;fill:none}` 即此风格）。圆点 `·`、箭头 `→`、`▾` 不算 emoji，可用。
- **暖纸主题**：只用 §1 列出的 CSS 变量，不引新色板、不引阴影/圆角新规范。
- **信源边界不可越线**：知乎/猪八戒**本轮只读**，绝不画它们的「自动回答/自动接单」按钮——只有小红书有自动发。设计稿若给知乎/猪八戒画了自动发，即为越界。
- **默认演练**：一键发送默认 dryrun 态，设计必须让「没真发」可见，别画成默认就真发的样子。
- **增量不重做**：复用 V1 版式与命名（StatsStrip/QueueItem/DetailPanel/三 Tab），只长出上面 5 处。
