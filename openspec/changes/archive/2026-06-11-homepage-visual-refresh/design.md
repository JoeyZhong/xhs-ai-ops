# 设计 · 首页视觉刷新（暖纸视觉 + 操作台布局）

> 视觉基线（单一真相）：`.mockup/homepage-redesign.html`。本文件给 token 与结构规格，落地以 mockup 为准。

## 1. 方案由来

三套方向验收：
- **A 晨光暖纸**：米纸底 + 衬线大问候 + 大留白，克制、编辑感。≈ 现状精炼。
- **B 深色专业台**：近黑暖炭 + 状态灯 + 输入框光圈 + 状态信息条，操作台体感。
- **C 活力卡片**：纯白 + 三张彩色能力卡，明亮引导。

**选定：A 的视觉 × B 的布局**——暖纸克制的皮肤，套上操作台的信息结构。既不冷（不走 B 的深色），
又把"能力/状态"在首屏显性化（B 的灯条 + 状态脚注），比纯 A 更"专业、可见"。

## 2. Token（`globals.css` `:root`）

沿用现有暖调，新增/微调：

| token | 值 | 用途 |
|---|---|---|
| `--bg` | `#f7f4ec`（现 `#faf9f5` 微暖） | 纸底（可保持现值，二选一，落地时定） |
| `--ring` | `rgba(232,68,90,.22)` | 输入框聚焦品牌红柔光圈 |
| `--agent-intel` | `#1d9e75`（teal） | 采集灯 |
| `--agent-analyst` | `#c98a1a`（amber） | 分析灯 |
| `--agent-content` | `#d4537e`（pink） | 生成灯 |

`--brand #e8445a`、`--card-warm`、`--border2`、`--text1/2/3` 不变。`@theme inline` 注册新增色变量。

## 3. 首屏结构（EmptyState，自上而下）

> **铁律：全程不用 emoji。** 所有图标用 `lucide-react`（前端已装）线性图标；现有 EmptyState 的
> emoji（问候手势 / chips / 目标）一并替换掉，不只是不新增。彩色圆点、文字箭头 `→`、`▾` 不算 emoji，可用。

```
[衬线问候]              晚上好，示例品牌                （去手势 emoji，纯文字；Georgia serif）
[副标题]    一句话说出运营意图，主助手替你动态调度下面三个智能体协作完成   （微调文案）
[★ 智能体状态灯条]   ● 采集   ● 分析   ● 生成   · 3 个智能体就绪          （新增；B 布局；圆点非 emoji）
[输入框 hero]      暖卡 + [target]目标选择 + 品牌红发送 ＋ 聚焦柔光圈        （Composer hero，加 ring）
[建议气泡 ×4]   [search]采集 / [chart]分析 / [pencil]规划 / [calendar]发布   （emoji→lucide 图标）
[★ 状态脚注]   动态编排 采集→分析→生成  ·  真流式协调  ·  可暂停追问        （取代纯说明文案；B 布局）
```

- **灯条**：三个 7px 圆点（各智能体色，带 .28 透明度柔晕）+ 文案；恒定展示，颜色对齐对话流里 archetype 语义（intel/analyst/content）。
- **柔光圈**：`box-shadow: 0 0 0 4px var(--ring)` 叠加在现有暖卡上，仅 `hero` 变体、聚焦/常显皆可（mockup 取常显，落地建议聚焦态出现以免太抢）。
- **状态脚注**：以小圆点分隔的三段（编排 / 真流式 / 可暂停），替换原 `主助手会自动编排…` 整句，信息更"操作台"。

## 4. 无障碍

- 灯条颜色仅作强调，语义同时由**文字**（采集/分析/生成）承载，非纯色编码（色盲安全）。
- 柔光圈不替代浏览器默认焦点可见性；保留 textarea `:focus` 可辨识。
- 字号 ≥ 12px；暖纸底 + `--text2/3` 对比达 AA。

## 5. 不做（YAGNI）

- 不接入真实 agent 健康状态（灯条是恒真能力提示，非实时监控）。
- 不动对话流（活动态）、`docked` 变体、`/admin`、深色主题。
- 不新建组件文件——改动落在现有 `EmptyState.tsx` / `Composer.tsx` / `globals.css`。
